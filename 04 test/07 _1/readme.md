
# _1〜_9のブロック変数

```ruby
a = %w(a b c d)
a.each do |e|
  p e
end
```

上記の|e|はブロック変数(ブロックパラメータ)と言います。
これはある意味で「変数宣言」とも言えます。
rubyなのに変数宣言するの？
この変数宣言をせずに動かす方法が今回のお話です。
上記のものは下記のように表せます。

```ruby
a = %w(a,1 b,2 c,3 d,4)
a.each do
  p _1.split(',')
end
```

doの右にあった|e|を省略し、代わりに_1で呼び出しています。
hashやwith_indexなど複数のブロック変数がある場合は、順番に
_1, _2, _3となっていきます。
複数になると分かりづらくなっていくように思います。
また、ブロックがネストしていった場合、かなり混乱すると思います。
というか、私の実験では外側のブロック変数を呼ぶことができませんでした。
なので今のところ使っているのは、一つのブロックで、一つのブロック変数の場合で、
何がブロック変数に入るのかすぐに分かる場合にのみ使っています。

個人的にはアンダースコアではなくコロン(_1 -> :1)の方が良かったなという感想です。
シンボルに数字って使ってみたい。
